## 问题2：人气榜单是如何排序的

### 排序逻辑说明

人气榜单在首页 `content.vue` 中实现，通过三个标签切换：

```javascript
const selectTab = (index) => {
  contentData.selectTabIndex = index
  let sort = (index === 0 ? 'recent' : index === 1 ? 'hot' : 'recommend')
  // ...
  getThingList({sort: sort})
}
```

### 三种排序方式

1. **最新** (`recent`)
   - 按 `create_time` 创建时间降序
   - 显示最新发布的服务

2. **最热** (`hot`) ⭐ 这就是人气榜单
   - 按 `pv` (Page View 浏览量) 降序
   - 浏览量越高，排名越靠前
   - 显示"浏览 100 人次"等信息

3. **推荐** (`recommend`)
   - 按 `recommend_count` 推荐数降序
   - 显示被推荐最多的服务

### 浏览量如何增加

每次用户点击服务卡片查看详情时，后端应该自动增加 `pv` 值：

```java
// ThingController.java 的 detail 方法
@RequestMapping(value = "/detail", method = RequestMethod.GET)
public APIResponse detail(String id){
    Thing thing = service.getThingById(id);
    if (thing == null) {
        return new APIResponse(ResponeCode.FAIL, "未找到数据");
    }
    
    // 增加浏览量
    service.incrementPv(id);
    
    return new APIResponse(ResponeCode.SUCCESS, "获取成功", thing);
}
```

### 数据库字段说明

`b_thing` 表中的统计字段：
- `pv`: 浏览量（Page View）- 用于"最热"排序
- `recommend_count`: 推荐数 - 用于"推荐"排序
- `wish_count`: 心愿单数量
- `collect_count`: 收藏数量

### 如何在前端显示浏览量

在服务卡片上显示：

```vue
<div class="stats-info">
  <span class="view-count">
    <i class="el-icon-view"></i>
    浏览 {{ item.pv || 0 }} 人次
  </span>
  <span class="collect-count">
    <i class="el-icon-star-off"></i>
    收藏 {{ item.collectCount || 0 }}
  </span>
</div>
```

## 问题3：作为服务提供者，如何查看别人预约了我的服务

### 问题分析

当前系统：
- ✅ "我的预约"页面：显示我预约别人的服务（作为客户）
- ❌ 缺少：显示别人预约我的服务（作为服务提供者）

### 解决方案

#### 前端部分（已完成）

1. **创建新页面**：`received-orders-view.vue`
   - 位置：`web/src/views/index/user/received-orders-view.vue`
   - 功能：显示客户预约我服务的订单

2. **添加路由**：
   ```javascript
   {
     path: 'receivedOrdersView',
     name: 'receivedOrdersView',
     component: () => import('/@/views/index/user/received-orders-view.vue')
   }
   ```

3. **添加菜单项**：
   在用户中心侧边栏添加"收到的预约"菜单

4. **页面功能**：
   - 显示预约编号、预约时间
   - 显示客户信息（姓名、电话、地址、备注）
   - 显示预约的服务和时间段
   - 提供操作按钮：接受预约、拒绝预约、联系客户

#### 后端部分（需要添加）

需要在后端添加新的API接口，详细代码见 `web/tmp/后端需要添加的代码.java`

**核心逻辑**：
1. 获取当前用户ID
2. 查询该用户发布的所有Thing（服务）
3. 查询这些Thing的所有预约记录
4. 返回预约列表（包含客户信息）

**API接口**：
```
GET /api/appointment/received
```

**返回数据示例**：

### 使用流程

#### 作为服务提供者：
1. 登录系统
2. 在"家政资料"中完善个人信息
3. 系统会自动创建服务（Thing）
4. 客户预约后，点击"收到的预约"查看
5. 可以查看客户信息和预约详情
6. 可以接受、拒绝或联系客户

#### 作为客户：
1. 浏览首页或分类页面
2. 点击服务卡片查看详情
3. 点击"立即预约"按钮
4. 选择日期和时间段
5. 填写接收人信息
6. 提交预约
7. 在"我的预约"中查看预约状态
