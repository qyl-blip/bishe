```java
@Override
public void incrementPv(String id) {
    Thing thing = mapper.selectById(id);
    if (thing != null) {
        int currentPv = thing.getPv() != null ? Integer.parseInt(thing.getPv()) : 0;
        thing.setPv(String.valueOf(currentPv + 1));
        mapper.updateById(thing);
    }
}
```

## 核心逻辑说明

### 查询逻辑
1. 从TOKEN获取当前登录用户ID
2. 通过SQL JOIN查询：
   - `b_appointment` 预约表
   - `b_thing` 服务表（通过 `t.user_id = #{userId}` 筛选该用户发布的服务）
   - `b_time_slot` 时间段表
3. 返回所有预约了该用户服务的订单

### SQL关键点
```sql
WHERE t.user_id = #{userId}  -- 关键：通过服务的user_id来筛选
```

这样就能查询到：
- 客户预约了"我发布的服务"的所有订单
- 包含客户信息（姓名、电话、地址）
- 包含预约时间和服务详情

## 数据库表关系

```
b_thing (服务表)
├── id (服务ID)
├── user_id (服务提供者ID) ← 关键字段
└── title, price, location...

b_appointment (预约表)
├── id (预约ID)
├── thing_id (服务ID) → 关联 b_thing.id
├── user_id (预约者ID/客户ID)
└── receiver_name, receiver_phone, receiver_address...

查询逻辑：
通过 b_thing.user_id = 当前用户ID
找到该用户发布的所有服务
再通过 b_appointment.thing_id
找到这些服务的所有预约
```
