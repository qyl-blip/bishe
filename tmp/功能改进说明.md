### 1. 添加"收到的预约"功能

**问题**：作为服务提供者，无法查看客户预约了自己服务的订单

**解决方案**：
- ✅ 创建了新页面：`web/src/views/index/user/received-orders-view.vue`
- ✅ 添加了路由：`receivedOrdersView`
- ✅ 在用户中心侧边栏添加了菜单项："收到的预约"
- ✅ 页面功能：
  - 显示客户预约信息（客户姓名、电话、地址、备注）
  - 显示预约的服务和时间
  - 提供操作按钮：接受预约、拒绝预约、联系客户

**需要后端支持**：
需要在 `AppointmentController.java` 中添加新的API接口：

```java
/**
 * 获取服务提供者收到的预约列表
 * 根据用户发布的Thing来查询预约
 */
@Access(level = AccessLevel.LOGIN)
@RequestMapping(value = "/received", method = RequestMethod.GET)
public APIResponse getReceivedAppointments(HttpServletRequest request) {
    // 1. 从TOKEN获取当前用户ID
    String token = request.getHeader("TOKEN");
    if (!StringUtils.hasText(token)) {
        return new APIResponse(ResponeCode.FAIL, "用户未登录");
    }
    
    User user = userService.getUserByToken(token);
    if (user == null) {
        return new APIResponse(ResponeCode.FAIL, "用户未登录");
    }
    
    try {
        String userId = user.getId().toString();
        // 2. 查询该用户发布的所有Thing
        List<Thing> userThings = thingService.getUserThing(userId);
        
        if (userThings.isEmpty()) {
            return new APIResponse(ResponeCode.SUCCESS, "获取成功", Collections.emptyList());
        }
        
        // 3. 获取这些Thing的所有预约
        List<String> thingIds = userThings.stream()
            .map(t -> String.valueOf(t.getId()))
            .collect(Collectors.toList());
        
        List<Appointment> appointments = appointmentService.getAppointmentsByThingIds(thingIds);
        
        return new APIResponse(ResponeCode.SUCCESS, "获取成功", appointments);
    } catch (Exception e) {
        logger.error("获取收到的预约列表失败 - 用户ID: {}, 错误: {}", user.getId(), e.getMessage(), e);
        return new APIResponse(ResponeCode.FAIL, "获取预约列表失败");
    }
}
```

还需要在 `AppointmentService` 中添加方法：
```java
List<Appointment> getAppointmentsByThingIds(List<String> thingIds);
```

### 3. 人气榜单排序逻辑说明

**当前实现**：
人气榜单是根据 `b_thing` 表的 `pv` 字段（浏览量）来排序的。

**数据字段说明**：
- `pv`: 浏览量（Page View）- 每次有人查看服务详情时+1
- `recommend_count`: 推荐数
- `wish_count`: 心愿单数量
- `collect_count`: 收藏数量

**排序规则**：
在首页 `content.vue` 中，点击"最热"标签时：
```javascript
let sort = (index === 0 ? 'recent' : index === 1 ? 'hot' : 'recommend')
```
- 最新：按创建时间降序
- 最热：按 `pv` 浏览量降序
- 推荐：按 `recommend_count` 降序

**如何增加浏览量**：
每次用户点击服务卡片进入详情页时，后端应该自动增加该服务的 `pv` 值。

需要在 `ThingController.java` 的 `detail` 方法中添加：
```java
@RequestMapping(value = "/detail", method = RequestMethod.GET)
public APIResponse detail(String id){
    Thing thing = service.getThingById(id);
    if (thing == null) {
        return new APIResponse(ResponeCode.FAIL, "未找到数据");
    }
    
    // 增加浏览量
    service.incrementPv(id);
    
    return new APIResponse(ResponeCode.SUCCESS, "获取成功", thing);
}
```

## 数据库字段说明

### b_thing 表关键字段：
- `id`: 服务ID
- `title`: 服务标题
- `price`: 价格（元/小时）
- `pv`: 浏览量
- `recommend_count`: 推荐数
- `wish_count`: 心愿单数
- `collect_count`: 收藏数
- `classification_id`: 分类ID（1=育儿嫂, 2=月嫂, 3=保洁, 4=护工）
- `status`: 状态（0=审核通过, 1=待审核, 2=已拒绝）
- `user_id`: 服务提供者ID

### b_appointment 表关键字段：
- `id`: 预约ID
- `appointment_number`: 预约编号
- `thing_id`: 服务ID
- `user_id`: 预约者ID（客户）
- `appointment_date`: 预约日期
- `slot_id`: 时间段ID
- `receiver_name`: 接收人姓名
- `receiver_phone`: 接收人电话
- `receiver_address`: 服务地址
- `status`: 状态（0=待服务, 1=已完成, 2=已取消）

## 使用说明

### 作为客户（预约服务）：
1. 浏览首页或分类页面
2. 点击服务卡片查看详情
3. 点击"立即预约"按钮
4. 选择日期和时间段
5. 填写接收人信息
6. 提交预约
7. 在"我的预约"中查看预约状态

### 作为服务提供者（接收预约）：
1. 在"家政资料"中完善个人信息
2. 系统会自动创建服务（Thing）
3. 客户预约后，在"收到的预约"中查看
4. 可以接受、拒绝或联系客户
5. 完成服务后更新预约状态
