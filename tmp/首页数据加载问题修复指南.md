# 首页数据加载问题修复指南

## 问题描述
1. 首页数据加载不出来，显示空白
2. 点击"月嫂"、"特惠保洁"等分类按钮没有反应

## 问题原因分析
1. 数据库中可能没有Thing数据
2. 分类表(b_classification)可能没有数据
3. 数据的status字段可能不是'0'（审核通过状态）

## 常见问题排查

### 问题1：API返回空数组
**原因**：数据库中没有status='0'的数据
**解决**：检查Thing表的status字段，确保有数据的status='0'

```sql
-- 查看status分布
SELECT status, COUNT(*) FROM b_thing GROUP BY status;

-- 如果都是'1'，批量更新为'0'
UPDATE b_thing SET status='0' WHERE status='1';
```

### 问题2：分类筛选不工作
**原因**：classification_id字段为NULL或不匹配
**解决**：检查并修复classification_id

```sql
-- 查看classification_id分布
SELECT classification_id, COUNT(*) FROM b_thing GROUP BY classification_id;

-- 确保所有数据都有classification_id
SELECT COUNT(*) FROM b_thing WHERE classification_id IS NULL;
```

### 问题3：图片不显示
**原因**：cover字段的图片文件不存在
**解决**：这是正常的，因为我们使用的是假图片地址。你需要：
1. 准备真实的图片文件
2. 上传到服务器的upload/image目录
3. 更新数据库中的cover字段

```sql
-- 查看图片地址
SELECT id, title, cover FROM b_thing LIMIT 10;
```
