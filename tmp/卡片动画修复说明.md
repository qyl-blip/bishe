# 卡片动画修复说明

## 问题原因

所有页面的`<style>`标签都使用了`scoped`属性，这导致CSS选择器无法正确应用到动态生成的子元素上。

## 解决方案

使用Vue 3的`:deep()`伪类选择器来穿透scoped样式作用域。

## 已修复的文件

### 1. LocalService.vue (附近家政)
```css
:deep(.card-col) {
  animation: cardFadeIn 0.6s cubic-bezier(0.34, 1.56, 0.64, 1) backwards !important;
}

:deep(.card-col:nth-child(1)) { animation-delay: 0.1s !important; }
:deep(.card-col:nth-child(2)) { animation-delay: 0.15s !important; }
/* ... 以此类推 */
```

## 动画效果

每张卡片会：
1. 从透明变为不透明（淡入）
2. 从下方20px位置向上移动到原位
3. 从95%缩放到100%
4. 每张卡片延迟0.05秒，形成依次出现的效果

## 技术说明

### 为什么需要:deep()？

Vue的`scoped`样式会给每个元素添加唯一的data属性（如`data-v-xxxxx`），CSS选择器也会被修改以匹配这些属性。但是：

- 动态生成的子组件元素不会自动获得这些属性
- 使用`:deep()`可以让选择器穿透到子组件
- `:deep()`是Vue 3推荐的方式（替代了Vue 2的`::v-deep`）

### 为什么使用!important？

确保动画样式的优先级高于其他可能的样式规则，避免被覆盖。
