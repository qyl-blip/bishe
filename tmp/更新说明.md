# 更新说明

## 1. 数据库更新

### 添加月嫂服务数据
执行SQL文件：`web/tmp/add_nanny_data.sql`

这个文件包含：
- 20条月嫂服务数据，覆盖北京、上海、广州、深圳、成都、杭州等城市
- 每条数据包含：姓名、描述、价格、年龄、地区、联系方式等
- `classification_id=2` 表示月嫂服务分类
- 分数在4.7-5.0之间，模拟真实评分

### 执行方式
```sql
-- 在MySQL中执行
source /path/to/web/tmp/add_nanny_data.sql;

-- 或者直接复制SQL内容到数据库管理工具执行
```

## 2. 前端组件更新

### 新增统一顶部栏组件
文件：`web/src/views/index/components/TopBar.vue`

功能：
- 第一行：Logo、搜索框、登录/用户信息

1. `web/src/views/local/NannyService.vue`
   - 使用TopBar组件替代原Header和定位栏

2. `web/src/views/local/LocalService.vue`
   - 使用TopBar组件替代原Header和定位栏
   - 支持城市切换后自动刷新数据

## 3. 分类说明

数据库中的`classification_id`字段用于区分服务类型：
- `1` = 护育服务
- `2` = 月嫂服务（新增）
- `3` = 家政服务
- `4` = 特惠服务

## 4. 使用说明

### 前端测试
1. 启动前端项目
2. 访问"月嫂"页面，应该只显示月嫂服务（classification_id=2）
3. 访问"附近家政"页面，应该只显示家政服务（classification_id=3）
4. 点击顶部的"切换城市"按钮，选择不同城市
5. 数据会根据选择的城市自动筛选

### 后端API要求
确保后端API `/api/thing/list` 支持以下参数：
- `city`: 城市名称
- `classificationId`: 分类ID
- `sort`: 排序方式（可选）

返回格式应该是：
```json
{
  "data": [
    {
      "id": 1,
      "title": "服务标题",
      "price": "380",
      "score": "4.9",
      "city": "北京",
      "classification_id": 2,
      ...
    }
  ]
}
```
